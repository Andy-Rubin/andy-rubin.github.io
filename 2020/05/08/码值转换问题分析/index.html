码值转换问题分析

2020-05-08
字数统计: 684   |   阅读时长≈ 2 分钟

在工作中经常会遇到这样的问题：
一些操作系统、主CPU或者上位机软件只能接收一定区间的数据，比如[6000,30000]之间的数，如果不在这一区域就会显示质量码出错的提示，
因为在实际项目中遇到的都是线性数据，比如4ma对应的是4800的码值，20ma对应的是31200码值，而下位机实际对应的码值范围是[68,38699],也是有着相同的线性逻辑。

如何将实际码值区间的数据映射到上位机区间的数据且对应的输入线性关系不变呢？

下面是实际的例子：

有块AD的板卡测量4~20ma的输入信号，输出结果是在10352 ~ 52352之间，成线性关系。

但是上位机显示的码值范围是6000~30000之间，内部逻辑也是线性，4ma对应6000码值，30000对应20ma码值。

那么它们之间映射关系是，
4ma输入后，下位机实际码值是10352，但是上位机输出码值是6000
20ma输入后，下位机实际码值是52352，但是上位机输出码值是52352， 总体呈线性关系

那么输入 8ma呢？上位机输出显示多少？

因为是线性的，令码值为因变量，输入的信号为自变量，所以可以计算出实际码值的线性曲线公式：y=kx+b， d1（4，10352），d2（20，52352）
不难算出，实际码值公式：y=2625x-148

这样就可以求出8ma的实际码值：y=2625 * 8 - 148 = 20852,
因为是线性比例，那么增加幅度为，20852 - 10352（基值） = 10500
增加比例为 10500 / (52352-10352) = 0.25;
那么 8ma 对应上位机数值：
6000+（30000-6000） * 0.25 = 12000

这是在知道输入ma的时候，但是很多时候下位机是个黑盒，内部只知道实际码值，比如在不知道输入信号前提下，
只知道实际产生码值是 36000， 那么在上位机显示多少？
分析同上，
增大了多少幅度？ 36000 - 10352 = 25648
增大比例多少？ 25648 / (52352-10352) = 0.61067
因为比例关系上位机下位机是同样的。
那么 就可以知道 这个输入信号在上位机显示： 6000 + （30000 - 6000）* 0.61067 = 20656

分别用变量代替公式里每个数值：
A(下位机底限)，B(下位机上限)
C(上位机底限)，D(上位机上限)
X(下位机输入的码值)
Y(上位机输出显示的码值)

那么公式为：
Y=(X-A) / (B-A) * (D - C) + C

即 Y=(X-10352) / (52352-10352) * (30000 - 6000) + 6000
在实际编码时最好将每个数值都计算出来，最后只呈现y=kx+b的形式。
